# 직원 업무 관리 툴 웹사이트 기획서

## 1. 개요

### 1.1 목적
- 팀별·개인 업무를 한 곳에서 관리
- 월/주/일 단위 스케줄 조회 및 관리
- 대표님의 전사 업무 현황 모니터링

### 1.2 대상 팀
| 팀명 | 설명 |
|------|------|
| 마케팅팀 | 마케팅 캠페인, 광고, SNS 등 |
| PM(기획)팀 | 프로젝트 기획, 일정·리소스 관리 |
| CS팀 | 고객 응대, 문의·민원 처리 |
| 디자인팀 | UI/UX, 그래픽, 영상 등 |

---

## 2. 사용자 역할 및 권한

### 2.1 역할 정의

| 역할 | 설명 | 권한 |
|------|------|------|
| **대표(관리자)** | 전사 업무 조회·관리 | • 전체 팀/전체 직원 스케줄 조회<br>• 월/주/일 단위 전사 캘린더<br>• 팀·직원 추가/수정/삭제<br>• 공지·일정 등록 |
| **팀원** | 본인 업무만 관리 | • 본인 월/주/일 스케줄 조회·수정<br>• 본인 업무 등록·수정·삭제<br>• 본인 팀 일정만 조회(선택) |

### 2.2 로그인·계정
- **개인 계정**: 아이디(이메일 또는 사번) + 비밀번호
- 최초 비밀번호는 관리자가 발급 후, 첫 로그인 시 변경 유도
- 비밀번호 찾기(이메일 인증 또는 관리자 초기화) 지원 권장

---

## 3. 핵심 기능

### 3.1 캘린더·스케줄 뷰

#### 3.1.1 월간 스케줄 (한눈에 보기)
- **표시**: 한 달 전체를 그리드 형태로 표시
- **대표님 뷰**: 팀별 색상 구분 또는 필터로 팀 선택 가능, 전 직원 일정 표시
- **직원 뷰**: 본인 일정만 표시
- **표시 정보**: 일정명, 담당자(대표 뷰 시), 팀, 간단 라벨(진행중/완료/예정 등)
- **클릭**: 해당 일정 클릭 시 상세 보기/수정(권한에 따라)

#### 3.1.2 주간 스케줄
- **표시**: 해당 주 월~일, 시간대별(선택) 또는 일별 요약
- **대표님 뷰**: 팀/직원 필터, 주간 전사 일정
- **직원 뷰**: 본인 주간 일정
- **기능**: 드래그로 일정 이동, 새 일정 추가

#### 3.1.3 일간 스케줄 (하루 일과)
- **표시**: 선택한 날짜의 시간대별 일정·업무 리스트
- **대표님 뷰**: 전 직원 또는 팀별 오늘 일과
- **직원 뷰**: 본인 오늘 일과
- **기능**: 
  - 업무 추가(제목, 설명, 시간, 우선순위, 상태)
  - 체크리스트 형태의 할 일 관리
  - 완료/미완료 표시

### 3.2 업무(일정) 관리

- **항목 예시**: 제목, 설명, 담당자, 팀, 시작일·종료일(또는 시간), 우선순위, 상태(예정/진행중/완료/보류), 연관 프로젝트(선택)
- **반복 일정**: 매일/매주/매월 반복 옵션
- **알림**: 마감 또는 시작 전 알림(이메일/앱 내 알림 중 선택)

### 3.3 대표님 전용 기능

- **대시보드**: 
  - 오늘/이번 주 전사 일정 요약
  - 팀별 진행 중인 업무 수, 완료율(선택)
- **전체 캘린더**: 월/주/일 뷰에서 팀·직원 필터
- **직원·팀 관리**: 팀 생성/수정, 직원 계정 생성/비밀번호 초기화, 팀 배정
- **공지/회사 일정**: 전사 공지, 휴무일, 회의 등 등록 후 전체 캘린더에 표시

## 4. 화면 구성 제안

### 4.1 공통
- **상단**: 로고, 메뉴(캘린더/일과/설정 등), 알림, 로그아웃
- **사이드바(선택)**: 월/주/일 전환, 팀 필터(대표님), 오늘로 이동

### 4.2 주요 화면
1. **로그인**: 아이디, 비밀번호, (선택) 비밀번호 찾기
2. **메인 캘린더**: 뷰 전환(월/주/일), 캘린더 그리드, 일정 클릭 시 상세/수정
3. **일정 상세/등록**: 모달 또는 별도 페이지로 일정 CRUD
4. **일과(오늘)**: 리스트 + 체크리스트 형태
5. **대표님 대시보드**: 요약 카드 + 팀별/전체 캘린더 링크
6. **설정**: 비밀번호 변경, (대표님) 팀·직원 관리

---

## 5. 데이터 구조 (개념)

### 5.1 주요 엔티티
- **사용자**: 아이디, 비밀번호(암호화), 이름, 이메일, 팀ID, 역할(대표/팀원)
- **팀**: 팀ID, 팀명(마케팅/PM/CS/디자인 등)
- **일정/업무**: 일정ID, 제목, 설명, 담당자ID, 팀ID, 시작일시, 종료일시, 상태, 우선순위, 반복규칙, 생성자
- **공지/회사일정**: 제목, 내용, 대상(전체/팀), 일자, 등록자

### 5.2 권한 규칙
- 일정 조회: 대표 → 전체, 팀원 → 본인
- 일정 수정/삭제: 대표 → 전체, 팀원 → 본인 담당 일정만

## 6. 기술·구현 시 고려사항

### 6.1 기술 스택 예시
- **프론트**: React / Vue / Next.js 등 SPA + 반응형
- **백엔드**: Node.js(Express 등) 또는 Next.js API Routes
- **DB**: PostgreSQL / MySQL + 일정용 쿼리 최적화
- **인증**: JWT + httpOnly 쿠키 또는 세션
- **캘린더 UI**: FullCalendar, react-big-calendar 등 라이브러리 검토

### 6.2 보안
- 비밀번호 해시 저장(bcrypt 등)
- HTTPS 필수
- 역할에 따른 API 접근 제어(미들웨어)
- 개인정보(이름, 이메일) 접근 권한 제한

### 6.3 확장 시 고려
- 팀장 역할(본인 팀만 조회) 추가
- 프로젝트 단위 그룹핑
- 간단한 리포팅/통계
- 모바일 대응 또는 PWA

## 8. 정리

- **팀**: 마케팅 / PM / CS / 디자인 4개 팀
- **스케줄**: 월간(한눈에) → 주간 → 일간(하루 일과) 모두 제공
- **대표님**: 전사 스케줄 조회·팀/직원 관리
- **직원**: 개인 아이디·비밀번호로 본인 업무만 관리

이 기획서를 바탕으로 요구사항을 더 구체화하시고, 원하시면 다음 단계로 **기술 스택 선정**이나 **화면 와이어프레임**까지 이어서 정리할 수 있습니다.
